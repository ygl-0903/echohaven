# 余音（echohaven）功能说明

本文档描述当前版本**已实现**的能力，便于使用与二次开发对照。

---

## 1. 产品定位

- **中文名**：余音  
- **英文名 / 包名**：echohaven  
- **形态**：桌面应用（Tauri 2），离线优先。  
- **不做**：云端账号、自动同步、浏览器自动填表、团队共享（当前版本未实现）。

---

## 2. 技术栈（实现层面）

| 层级 | 技术 |
|------|------|
| 壳与系统集成 | Tauri 2 |
| 界面 | React 19 + TypeScript + Vite 6 |
| UI 组件 | Mantine 8（AppShell、表单、通知等） |
| 路由 | react-router-dom（`/` 首页、`/items/new`、`/items/:id`、`/settings`） |
| 样式辅助 | Tailwind CSS v4（少量全局样式） |
| 状态 | Zustand |
| 敏感逻辑与文件 IO | Rust（`src-tauri`） |
| 系统集成插件 | 系统通知、文件对话框、剪贴板、使用系统浏览器打开链接 |

更细的存储与文件格式见：[storage-design.md](./storage-design.md)、[vault-format-v1.md](./vault-format-v1.md)。

## 3. 金库与密码学

- **默认金库路径**：系统用户数据目录下 `echohaven/echohaven.vault`（由 Rust 侧解析，前端可显示完整路径）。  
- **主密码**：创建时至少 8 位；**无法找回**，丢失主密码即无法解密金库。  
- **密钥派生**：Argon2id；创建金库时可勾选「快速创建」以使用较低 KDF 成本（适合开发机，正式使用建议关闭）。  
- **金库加密**：AES-256-GCM；每次保存使用新的随机 nonce；文件头包含盐与 Argon2 参数，便于备份文件在别机用同一主密码解锁。  
- **文件标识**：魔数 `EHVN`，版本字节当前为 v1。  
- **单元测试**：`vault` 模块含加解密往返、错误密码拒绝等测试（`cargo test`）。

## 4. 解锁、锁定与会话

- **首次使用**：无默认金库文件时，引导设置主密码并创建金库。  
- **已有金库**：主界面为解锁；可选切换到「新建金库」标签（若当前所选路径已存在文件，会提示需先备份/删除或走导入流程）。  
- **金库路径**：解锁/创建时可指定**任意 `.vault` 路径**（输入框、**打开…** 选已有文件、**另存为…** 新建路径）；留空则使用默认路径。本机会记住**上次成功打开的路径**（优先预填），并维护**最近使用路径**列表（`localStorage`，最多约 12 条），便于多金库、U 盘等场景。  
- **主密码修改**：**设置 → 安全与备份** 中可修改主密码（需输入当前密码；新密码至少 8 位）。成功后立即重加密写盘并更新内存中的会话密钥，**无需重新解锁**。当前密码错误会计入解锁同类失败次数并可能触发冷却。  
- **解锁失败保护**：连续失败约 5 次后触发约 **30 秒**冷却，防止本地快速试错。  
- **锁定**：顶栏「锁定」清空内存会话并返回解锁界面；应用重启后会话为空，需重新输入主密码。  
- **闲置自动锁定**：设置中可选 5 / 15 / 30 / 60 分钟或关闭；依据键盘与指针操作刷新「最后活动时间」，超时后自动锁定。偏好保存在浏览器 `localStorage`（仅前端策略，不涉及金库内容）。

## 5. 密码条目管理

每条目支持字段：

- 标题、用户名、密码、网址、备注  
- **标签**（`TagsInput` 输入；可与「标签预设」一键叠加）  
- **提醒**（可选）：类型（定期轮换 / 到期 / 自定义）、到期日（`YYYY-MM-DD`）、是否启用；服务端记录「某日已通知」以避免同日重复推送（见下文提醒流程）

**标签预设**（加密金库内 `tagPresets`）：

- 在 **设置 → 标签与分类** 维护名称与可选颜色；保存后写入金库。  
- **密码库页**可按预设名称**筛选**条目；编辑条目时可点预设快速加入 `tags`。

界面与导航：

- **AppShell**：顶栏为应用名、当前金库文件名（悬停可看完整路径）、**锁定**；左侧为**导航**（密码库、新建密码、设置），宽度可拖拽调整（本机记忆）。搜索与标签筛选**不在侧栏**，而在密码库主页面。  
- **密码库**（`/`）：主区域为**表格**（标题、用户名、网址/摘要、标签、提醒图标）；顶部为搜索框与标签筛选；点击行进入详情。  
- **新建**（`/items/new`）：独立表单页，保存后跳转该条详情。  
- **详情**（`/items/:id`）：查看与编辑。  
- **设置**（`/settings`）：多标签页（标签与分类、外观配色、安全与备份、关于）。

其他界面能力：

- **搜索**：在密码库页按标题、用户名、网址、备注、标签全文过滤。  
- **新建 / 保存 / 删除**：变更经 Rust 写回加密文件。删除前使用 **Tauri 原生确认框**（`plugin-dialog` 的 `ask`），避免 WebView 内 `window.confirm` 行为异常。  
- **显示/隐藏密码**、**复制密码**：复制后约 **30 秒**尝试将剪贴板写空（受操作系统与其他应用行为影响，不保证绝对清空）。  
- **密码生成器**：长度、字符类（小写/大写/数字/符号）、「易读」模式；一键生成并填入。  
- **网址**：可在系统默认浏览器中打开（opener 插件）。

## 6. 提醒与系统通知

- **逻辑**：仅对「启用提醒」且「到期日 ≤ 当天」且「当日尚未记录为已通知」的条目视为待提醒。  
- **流程**：  
  1. 前端周期调用 `scan_due_reminders` 拉取待提醒列表（只读，不改金库）。  
  2. 若系统通知权限已授予，则逐条 `sendNotification`；**仅在成功展示通知后**再调用 `ack_due_reminders` 写入 `lastNotifiedOn`，避免「没权限却标记已提醒」导致用户永远收不到提醒。  
- **权限未授予**：不会确认已提醒，下次周期会再次尝试（用户授权后可收到积压提醒）。  
- **限制**：当前未实现「点击通知跳转到指定条目」的深链；通知文案包含条目标题与到期日。

## 7. 备份与导入

- **导出加密备份**：设置中选择保存路径，将**当前会话正在使用的金库文件原样复制**为 `.ehv` / `.vault` 等（仍为加密，无明文）。  
- **导出条目文档（Markdown）**：设置中可将当前条目导出为 **UTF-8 Markdown**；默认**不包含**明文密码，可选开启「包含密码」——此时文件等同敏感明文，需自行保管。  
- **整库导入（替换）**：选择备份文件并输入**该备份的主密码**；校验通过后，用备份内容**覆盖当前正在使用的金库文件**（与解锁时选定的路径一致，不限于「仅默认路径」）。  
- **导入前自动备份**：若当前金库文件在磁盘上已存在，会先在**同目录**生成带时间戳的副本，文件名形如 `{原主名}.pre-import-YYYYMMDD-HHMMSS.vault`，再执行覆盖；成功提示中可含该备份路径（`UnlockInfo.preImportBackupPath`）。  
- **导入前落盘**：确认导入前会调用 `save_vault_disk`，尽量将内存中未持久化的修改先写入当前金库。  
- **导入对话框**：可展示所选备份文件的**大小与本地修改时间**（`stat_vault_file`），并提示当前已加载条目数，便于心理确认。  
- **风险提示**：整库替换；操作前有文案说明，需在对话框中确认。

## 8. 无障碍与界面

- 主要区域使用 `<main role="main">`；加载态使用 `role="status"` / `aria-live`。  
- Mantine 表单控件带内置标签与焦点样式。  
- **外观**：默认深色；可在设置中切换浅色 / 深色 / 跟随系统，方案保存在本机 `localStorage`（键名由 Mantine `colorSchemeManager` 管理）。

## 9. 后端命令一览（Tauri IPC）

前端通过 `invoke` 调用的命令名称（Rust 侧 snake_case，参数多为 camelCase 传参）包括：

| 命令 | 作用 |
|------|------|
| `get_default_vault_path` | 返回默认金库绝对路径 |
| `vault_file_exists` | 判断金库文件是否存在（可选自定义路径） |
| `stat_vault_file` | 读取金库文件本地元数据（大小、修改时间），供导入前展示 |
| `create_vault` | 创建金库并解锁会话（可选自定义路径） |
| `unlock_vault` | 解锁会话（可选自定义路径） |
| `change_master_password` | 已解锁状态下修改主密码并重加密写盘 |
| `lock_vault` | 清除会话 |
| `is_unlocked` | 查询是否处于解锁状态 |
| `list_entries` | 列出全部条目 |
| `list_tag_presets` | 列出标签预设 |
| `save_tag_presets` | 替换并持久化标签预设 |
| `upsert_entry` | 新增或更新条目 |
| `delete_entry` | 按 id 删除 |
| `save_vault_disk` | 将会话数据重新加密写盘（当前实现中主要随 upsert/delete 自动持久化） |
| `export_vault` | 导出当前金库文件到指定路径 |
| `export_entries_document` | 导出条目为 Markdown 文档（可选是否含明文密码） |
| `import_vault_replace` | 从备份解密校验后，覆盖**当前会话金库路径**对应文件并刷新会话 |
| `scan_due_reminders` | 扫描待提醒条目（不写入） |
| `ack_due_reminders` | 确认已展示通知，更新条目提醒状态并保存 |

## 10. 构建与 CI

- **本地**：`npm run tauri dev` / `npm run tauri build`。  
- **CI**（GitHub Actions）：  
  - Ubuntu：安装依赖后执行 `npm ci` 与 `npm run build`。  
  - macOS：在 `src-tauri` 下执行 `cargo test`。

## 11. 已知边界与后续可扩展方向

- **密码库表格**当前为常规表格 + 区域滚动；单库条目极多时可再考虑虚拟滚动或分页。  
- **通知 → 指定条目**：尚未实现点击通知跳转/深链；文案含标题与到期日。  
- **系统睡眠/锁屏立即锁定**：可增强，当前主要依赖闲置计时锁定。  
- **剪贴板**：约 30 秒后尝试清空依赖系统与其他应用行为，**不能**作为强安全保证；跨设备剪贴板同步等场景亦不在控制范围内。  
- **AppShell 布局**：主内容区可单独滚动；根容器避免使用 `overflow: hidden` 以免在部分 WebView 中影响 `sticky` 顶栏显示（实现上已注意此点）。
